# 빌리지 유입로그 AI 에이전트 매뉴얼

> 카카오톡 문의 채팅에서 유입경로·문의장비·고객유형을 추출 → 스프레드시트 자동 기록

---

## 시스템 개요

빌리지 렌탈샵의 카카오톡 채널 문의를 AI가 분석하여, 마케팅 유입 데이터를 '빌리지 마케팅 1.0' 스프레드시트에 자동 기록하는 에이전트 시스템.

### 운영 방식 (2가지)

1. **캡쳐 전달 방식**: 재형님이 카톡 캡쳐를 Cowork에 보내면 → 에이전트가 이미지를 분석하여 유입 정보 추출 → API로 기록
2. **Chrome 직접 스캔**: Claude in Chrome으로 카카오톡 채널 관리자에 직접 접근 → 채팅 읽고 분석 → API로 기록

### 사용 도구
- 카카오톡 채널 관리자: `https://business.kakao.com/_xhPMls/chats`
- 구글 시트 (빌리지 마케팅 1.0): 유입로그 탭
- GAS API: `https://script.google.com/macros/s/AKfycbzT9asf-IaI2Q4C6pWqoLZm9PzwNQv6RIzbaGawu1UL-KnnhMszmCrXXG4aLIKjBxQY/exec`
- Claude in Chrome: 카카오톡 채널 관리자 직접 접근
- Cowork 대화창: 재형님과의 커뮤니케이션 (디스패치 알림)

---

## 방식 1: 캡쳐 전달

### 흐름
1. 재형님이 카톡 채팅 캡쳐를 Cowork에 전달
2. 에이전트가 이미지를 직접 분석 (별도 API 호출 없이 에이전트 자체 시각 분석)
3. 유입경로, 문의장비, 고객유형 추출
4. GAS API 호출하여 스프레드시트에 기록
5. 결과 보고

---

## 방식 2: Chrome 직접 스캔

### 흐름
1. 재형님이 "유입로그 정리해줘" 명령
2. Claude in Chrome으로 카카오톡 채널 관리자 접근
3. 채팅 목록 스캔 → 각 채팅방 진입 → 메시지 분석
4. 유입 정보 추출 후 GAS API로 기록
5. 결과 보고

### 채팅 목록 접근
- URL: `https://business.kakao.com/_xhPMls/chats`
- 개별 채팅방: `https://business.kakao.com/_xhPMls/chats/{채팅방ID}`

## 유입 정보 추출 기준

### 유입경로 판단

| 유입경로 | 키워드/패턴 |
|---------|------------|
| 네이버검색 | `네이버에서`, `검색해서`, `블로그 보고`, `네이버 검색` |
| 인스타그램 | `인스타에서`, `인스타그램`, `SNS`, `릴스 보고` |
| 당근마켓 | `당근에서`, `당근마켓` |
| 지인소개 | `소개받아서`, `친구가`, `지인`, `추천받아서` |
| 기타 | 위에 해당 없거나 유입경로 언급 없음 |

**판단 우선순위:**
1. 고객이 직접 언급 → 그대로 기록
2. 표준 양식(이름/연락처/구분)으로 바로 온 경우 → 대부분 네이버검색 (블로그 양식 따라옴)
3. 비정형 대화체 → 기타
4. 이전 대화 이력 있는 경우 → 재방문, 유입경로는 기타

### 문의장비 추출

- 카메라, 렌즈, 조명, 삼각대, 짐벌, 모니터, 무선장비 등 모든 장비 포함
- 여러 개면 쉼표 구분: `FX3, 24-70 GM, 600C`
- 약어 그대로 기록: `FX3`, `s3`, `2470gm2`
- 수량 생략 (어떤 장비에 관심있는지만 기록)
- 장비 미언급 → `미확인`

### 고객유형 판단

| 고객유형 | 판단 기준 |
|---------|----------|
| 재방문 | 이전 대화 이력 있음, `다시`, `또`, `전에도`, `재렌탈`, `추가 예약` |
| 신규 | 첫 대화, 이전 이력 없음, 재방문 힌트 없음 |

### 제외 대상 (유입로그에 기록 안 함)

- 반납 완료, 입금 확인, 단순 감사/확인
- 장비 파손/분실 보고, 도착 시간 연락
- 챗봇 자동응답 (`챗봇이 보냄`)
- 직원 메시지 (`빌리지님이 보냄`, `김준영님이 보냄`)

## GAS API 사용법

### 기록 (create)
```
GET {API_URL}?action=create&유입경로={값}&고객유형={값}&문의장비={값}&메모={값}
```

| 파라미터 | 필수 | 값 |
|---------|------|-----|
| action | O | `create` |
| 유입경로 | O | 네이버검색 / 인스타그램 / 당근마켓 / 지인소개 / 기타 |
| 고객유형 | O | 신규 / 재방문 |
| 문의장비 | O | 장비명 (쉼표 구분) |
| 메모 | X | 고객명, 특이사항 등 |
| 예약여부 | X | Y / N (기본 N) |

### 목록 조회
```
GET {API_URL}?action=list
GET {API_URL}?action=list&filter=pending   (미확정)
GET {API_URL}?action=list&filter=booked    (예약)
GET {API_URL}?action=list&filter=lost      (미예약)
```

### 수정
```
GET {API_URL}?action=update&row={행번호}&예약여부=Y&예약금액=500000
```

### 중복 방지
- 기록 전 `action=list`로 기존 목록 조회
- 같은 날짜 + 같은 장비 조합이 이미 있으면 스킵
- 메모에 고객명 기록하면 중복 판단에 도움
